DirBuster Demo

How to run the demo safely

This page shows how to start the local server and run directory brute force tools against it.

Start the demo server

  1. Ensure Python 3 is installed.
  2. From the project root, start the server: 
    python server.py --host 0.0.0.0 --port 8000
  3. Visit the site:
    • http://127.0.0.1:8000 (local)
    • http://YOUR_LAN_IP:8000 (from other devices on the same network)
  4. If you cannot reach it from other devices, allow Python through your firewall or open TCP port 8000.

Run directory brute-force tools

Gobuster (recommended):

gobuster dir -u http://YOUR_LAN_IP:8000 -w ./wordlists/demo-common.txt -t 30 -x html,txt,zip

DirBuster (GUI): Target URL = http://YOUR_LAN_IP:8000, Wordlist = ./wordlists/demo-common.txt, Threads = 30, File extensions = html,txt,zip.

Watch for HTTP 200/301/403 responses. You should find /admin/, /backup/, /private/ (403), /old/, /uploads/, and /robots.txt.

What this demonstrates

  • Security through obscurity is weak: hidden paths can be discovered.
  • Robots.txt leakage: sensitive paths sometimes get listed to dissuade crawlers.
  • Backups in web root: risk of downloading configs or old code.
  • 403 vs file access: /private/ returns 403, but files like /private/secrets.txt may still be reachable.
Only test on systems you own or have explicit permission to test.
